Design

System architecture

 

The Itchy AI web application system architecture is very minimal, only having two components directly relating to the web application: the webpage in the frontend and the web server in the frontend. The computer will send its webcam image to the webpage which will encode it and call the webserver’s API and receive the itch data back which will be displayed on the webpage for the user to see.

API

There are three API endpoints that are for: starting calibration, updating sensitivity, and detecting an itch.

Starting calibration:

·         URL route: /api/calibrate

·         Request method: POST

Updating sensitivity:

·         URL route: /api/update_sensitivity

·         Request method: POST

·         Body:

o   lip_sensitivity : float

o   eyebrow_sensitivity : float

Detecting an itch:

·         URL route: /api/detect

·         Request method: POST

·         Body:

o   image: string ( .png or .jpg image converted to a base 64 string)
